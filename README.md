# VPN-сервис: бот в Telegram + панель управления

Это готовый «скелет» твоего собственного VPN-сервиса. Объясняю всё простыми словами,
шаг за шагом, как для новичка. Не пугайся длины — это потому, что я ничего не пропускаю.

---

## 1. Что это вообще такое (на пальцах)

Представь, что у тебя есть **дверь в интернет** (твой сервер в Амстердаме с Amnezia VPN).
Чтобы войти в эту дверь, человеку нужен **ключ** (файл-конфиг WireGuard).

Эта программа — твой **автоматический вахтёр**:

- Клиент пишет твоему **боту в Telegram**, выбирает тариф и платит.
- Ты (или платёжная система) подтверждаешь оплату.
- Вахтёр сам **делает ключ**, **пускает клиента** на сервер и **присылает ему ключ**.
- Вахтёр следит за сроком: за день до конца — **напоминает**, а когда срок вышел — **закрывает дверь**.
- Ты можешь **ограничить скорость** каждому клиенту и **запретить протоколы** (например, торренты).
- Есть **панель в браузере**, где видно всех клиентов и можно всем управлять.

Всё это хранится в маленькой **базе данных** (один файл), где лежит история платежей и подписок.

### Из каких кусочков состоит программа

| Файл | Что делает (простыми словами) |
|------|-------------------------------|
| `bot.py` | Сам бот в Telegram: общается с клиентами и с тобой |
| `web_panel.py` | Панель в браузере для наблюдения и управления |
| `database.py` | «Тетрадка», где всё записано (клиенты, платежи, события) |
| `vpn_manager.py` | Делает ключи и пускает/выгоняет клиентов на сервере |
| `traffic_control.py` | Ограничивает скорость и протоколы каждому клиенту |
| `payments.py` | Принимает оплату (ручную или через ЮKassa) |
| `notifier.py` | Сам напоминает об окончании и закрывает доступ |
| `config.py` | Читает твои настройки из файла `.env` |
| `deploy/` | Скрипт установки и файлы для автозапуска |

## 2. Что нужно иметь заранее

1. Сервер с **Ubuntu 22.04** (он у тебя есть — VDsina, Амстердам).
2. На сервере уже стоит **Amnezia VPN** (тоже есть). Внутри это, скорее всего,
   **AmneziaWG** — это «WireGuard с маскировкой». Для нашей программы это то же самое,
   что WireGuard, только команда называется `awg` вместо `wg`.
3. Умение подключаться к серверу по **SSH** (через терминал или PuTTY).
4. Аккаунт в **Telegram** (на телефоне).

> Совет новичку: всё, что ниже, делается командами в чёрном окне терминала на сервере.
> Не бойся — просто копируй команды по очереди.

## 3. Шаг за шагом: установка

### Шаг 3.1. Зайди на сервер и скачай проект

```bash
# подключаемся к серверу (подставь свой адрес)
ssh root@АДРЕС_СЕРВЕРА

# идём в домашнюю папку и скачиваем проект
cd /root
git clone <АДРЕС_ЭТОГО_РЕПОЗИТОРИЯ> vpn-service
cd vpn-service
```

> Если git не настроен — можно просто скопировать файлы проекта в папку `/root/vpn-service`.

### Шаг 3.2. Запусти установщик

```bash
bash deploy/setup.sh
```

Он сам поставит Python, создаст изолированное окружение `.venv`, установит библиотеки
и включит «пересылку пакетов» (без неё VPN не раздаёт интернет). В конце он создаст файл
`.env` из шаблона — его мы сейчас заполним.

## 4. Создаём бота в Telegram (2 минуты)

1. В Telegram найди **@BotFather**, напиши `/newbot`.
2. Придумай имя и логин боту. BotFather выдаст **токен** — длинную строку вида
   `123456:ABC-DEF...`. Скопируй её.
3. Теперь узнай **свой Telegram ID**: напиши боту **@userinfobot**, он пришлёт число
   (например `123456789`). Это и есть твой ID администратора.

## 5. Узнаём данные VPN-сервера

Нам нужно 4 вещи от твоего WireGuard/AmneziaWG. Выполни на сервере:

```bash
# Имя VPN-интерфейса и публичный ключ сервера.
# Для обычного WireGuard:
sudo wg show
# Для AmneziaWG (если предыдущая команда не нашлась):
sudo awg show
```

Ты увидишь что-то вроде:
```
interface: wg0                 <- ИМЯ ИНТЕРФЕЙСА (запиши)
  public key: AbCd1234...=     <- ПУБЛИЧНЫЙ КЛЮЧ СЕРВЕРА (запиши)
  listening port: 51820        <- ПОРТ (запиши)
```

Ещё нужно:

```bash
# Внешний IP сервера (адрес, по которому к нему подключаются):
curl -4 ifconfig.me

# Имя интерфейса, который смотрит в интернет (обычно eth0 или ens3):
ip route get 8.8.8.8
# в ответе найди слово "dev", сразу после него — имя, например "dev eth0"
```

Запиши себе: **имя VPN-интерфейса**, **публичный ключ сервера**, **внешний IP:порт**,
**имя интернет-интерфейса**, и какая **подсеть** у VPN (обычно `10.8.0.0/24` — можно
подсмотреть в конфиге сервера `/etc/amnezia/amneziawg/awg0.conf` или `/etc/wireguard/wg0.conf`).

## 6. Заполняем настройки `.env`

Открой файл:

```bash
nano .env
```

И заполни (остальное можно оставить как есть):

```ini
TELEGRAM_BOT_TOKEN=сюда_токен_от_BotFather
ADMIN_IDS=сюда_твой_Telegram_ID

# тарифы: дни:цена через запятую
PLANS=30:150,90:400,180:700

# оплата: для старта оставь manual
PAYMENT_PROVIDER=manual
SBP_PHONE=+7 900 000-00-00
SBP_RECEIVER=Иван И., Сбербанк

# данные VPN-сервера (из шага 5)
WG_INTERFACE=wg0           # или awg0 для AmneziaWG
WG_CMD=wg                  # или awg для AmneziaWG
WG_SERVER_PUBLIC_KEY=ключ_сервера
WG_SERVER_ENDPOINT=ВНЕШНИЙ_IP:51820
WG_SUBNET=10.8.0.0/24
WG_DNS=1.1.1.1

# интернет-интерфейс (из шага 5)
WAN_INTERFACE=eth0

# ВАЖНО: пока тестируем — оставь false (правила только показываются, не применяются)
APPLY_TRAFFIC_RULES=false

# пароль панели — ОБЯЗАТЕЛЬНО смени
WEB_PANEL_USER=admin
WEB_PANEL_PASSWORD=придумай_сложный_пароль
```

Сохрани: нажми `Ctrl+O`, `Enter`, потом `Ctrl+X`.

## 7. Первый запуск (безопасный режим)

Пока `APPLY_TRAFFIC_RULES=false`, программа **ничего не меняет на сервере** — только
показывает в логах, какие команды она *выполнила бы*. Это специально, чтобы ты ничего
не сломал, пока разбираешься.

```bash
# включаем окружение и запускаем бота
source .venv/bin/activate
python bot.py
```

Теперь в Telegram напиши своему боту `/start`. Должно появиться меню. Проверь полный круг:

1. Нажми «🛒 Купить / продлить», выбери тариф.
2. Нажми «✅ Я оплатил».
3. Тебе (как админу) придёт сообщение с кнопками «Подтвердить / Отклонить».
4. Нажми «Подтвердить» — бот пришлёт тебе **файл-конфиг** `vpn.conf` и текст ключа.

Если всё это произошло — поздравляю, логика работает! 🎉
(Ключ пока «ненастоящий» в смысле трафика, потому что мы в безопасном режиме —
peer на сервере добавится только когда включишь `APPLY_TRAFFIC_RULES=true`.)

Останови бота сочетанием `Ctrl+C`.

## 8. Включаем «боевой» режим (управление сервером)

Когда убедился, что бот работает, можно разрешить программе реально управлять сервером:
добавлять клиентов, ограничивать скорость и протоколы.

1. В `.env` поставь:
   ```ini
   APPLY_TRAFFIC_RULES=true
   ```
2. Один раз настрой «корень» ограничителя скорости (на VPN-интерфейсе):
   ```bash
   source .venv/bin/activate
   python -c "import traffic_control as t; print('\n'.join(t.setup_root_qdisc()))"
   ```
3. Запусти бота снова — **обязательно от root** (чтобы он мог выполнять `wg`/`tc`/`nft`):
   ```bash
   python bot.py
   ```

Теперь при подтверждении оплаты программа:
- добавит клиента (peer) на сервер командой `wg set ...`;
- включит ему ограничение скорости из настроек (`DEFAULT_SPEED_LIMIT_MBIT`);
- пришлёт клиенту рабочий ключ.

### Как работает ограничение скорости и протоколов (коротко)

- **Скорость** ограничивает встроенная в Linux система `tc`. Каждому клиенту по его
  VPN-адресу выдаётся «своя труба» нужной ширины. Сейчас ограничивается **скачивание** —
  это самое заметное для клиента. Ограничение **отдачи** (upload) требует доп. настройки
  виртуального интерфейса IFB (см. раздел «Что можно улучшить»).
- **Протоколы** фильтрует `nftables`. Профиль клиента:
  - `all` — без ограничений;
  - `web` — только сайты (порты 80/443) и DNS, остальное блокируется;
  - `no_torrent` — всё, кроме типичных торрент-портов.

Профиль меняется в боте командой `/limit` или в веб-панели.

## 9. Веб-панель в браузере

В отдельном окне терминала (бот пусть работает в своём):

```bash
source .venv/bin/activate
python web_panel.py
```

Открой в браузере: `http://АДРЕС_СЕРВЕРА:8080`
Логин и пароль — те, что ты задал в `.env` (`WEB_PANEL_USER` / `WEB_PANEL_PASSWORD`).

В панели видно: всех клиентов, их сроки, трафик, скорость и профиль; журнал платежей;
последние события. Кнопками можно **продлить** подписку, **поменять лимиты** и **отключить** клиента.

> Подтверждать оплаты удобнее в боте — там клиенту сразу уходит ключ.
> Панель — для наблюдения и ручного управления.

> Безопасность: открой порт `8080` только для себя (по своему IP) через firewall, либо
> заходи через SSH-туннель. Не оставляй панель открытой всему интернету со слабым паролем.

## 10. Команды администратора в боте

Пиши их боту (работают только для ID из `ADMIN_IDS`):

| Команда | Что делает |
|---------|-----------|
| `/pending` | Показать оплаты, ждущие подтверждения |
| `/users` | Список всех подписок |
| `/extend 123456789 30` | Продлить клиенту подписку на 30 дней |
| `/limit 123456789 100 50 all` | Лимиты: 100 ГБ, 50 Мбит/с, профиль `all` |

## 11. Чтобы работало постоянно (автозапуск)

Сейчас бот работает, только пока открыт терминал. Чтобы он запускался сам и не падал —
настроим автозапуск через systemd.

1. Открой `deploy/vpn-bot.service` и `deploy/vpn-panel.service`, проверь, что путь
   `/root/vpn-service` совпадает с твоим.
2. Установи службы:
   ```bash
   sudo cp deploy/vpn-bot.service /etc/systemd/system/
   sudo cp deploy/vpn-panel.service /etc/systemd/system/
   sudo systemctl daemon-reload
   sudo systemctl enable --now vpn-bot
   sudo systemctl enable --now vpn-panel
   ```
3. Смотреть логи:
   ```bash
   journalctl -u vpn-bot -f
   journalctl -u vpn-panel -f
   ```

Теперь бот и панель работают сами, перезапускаются при сбое и стартуют после перезагрузки.

## 12. Про оплату — честно и по-простому

### Вариант A. Ручное подтверждение (рекомендую для старта) — `PAYMENT_PROVIDER=manual`
Клиент переводит тебе деньги по **СБП на номер телефона** (или на кошелёк) и жмёт
«Я оплатил». Тебе приходит уведомление — ты смотришь в приложении банка, что деньги
пришли, и жмёшь «Подтвердить». Ключ уходит клиенту **автоматически**. Это работает
**сразу**, без договоров. Минус — подтверждаешь руками.

### Вариант B. Автоматический приём СБП — `PAYMENT_PROVIDER=yookassa`
Чтобы оплату подтверждала **сама система**, нужен сервис-посредник (эквайринг):
**ЮKassa**, Точка, Tinkoff и т.п. Для этого нужен договор и ключи. Когда получишь ключи —
впиши `YOOKASSA_SHOP_ID` и `YOOKASSA_SECRET_KEY` в `.env`, поставь `PAYMENT_PROVIDER=yookassa`.
Код-заготовка уже есть в `payments.py`.

> Важно: «полностью автоматически принимать перевод по номеру телефона» без посредника
> и без доступа к банковскому API **нельзя** — банк не отдаёт уведомления о входящих
> переводах в чужую программу. Поэтому либо ручное подтверждение, либо эквайринг.

### Вариант C. Antarctic Wallet — `PAYMENT_PROVIDER=antarctic`
Публичного API для **приёма** платежей у этого кошелька я не нашёл. Поэтому сейчас это
работает как ручной режим: бот показывает адрес кошелька (`ANTARCTIC_WALLET` в `.env`),
клиент платит, ты подтверждаешь. Если у Antarctic появится API — его подключают в
`payments.py` рядом с заготовкой ЮKassa.

## 13. Частые проблемы (FAQ)

- **Бот не отвечает.** Проверь `TELEGRAM_BOT_TOKEN`. Запусти `python bot.py` и читай ошибки.
- **«Не хватает настроек в .env».** Программа сама подскажет, что именно не заполнено.
- **Команда `wg`/`awg` не найдена.** Поставь правильное значение `WG_CMD` (для Amnezia это `awg`).
- **Клиент подключается, но нет интернета.** Проверь, что включён `ip_forward` (его включает
  `deploy/setup.sh`) и что на сервере есть правило NAT (`MASQUERADE`) — обычно его создаёт
  сама Amnezia. Проверь: `sudo iptables -t nat -L POSTROUTING -n`.
- **Скорость не ограничивается.** Убедись, что `APPLY_TRAFFIC_RULES=true`, бот запущен
  от root, и ты один раз выполнил `setup_root_qdisc` (шаг 8.2).
- **Не пускает в панель.** Логин/пароль берутся из `.env`. Поменял `.env` — перезапусти панель.

## 14. Что можно улучшить дальше (когда освоишься)

- **Учёт трафика** (сколько клиент скачал) и автоблокировка при превышении лимита —
  данные можно брать из `wg show <iface> transfer` и складывать в базу
  (поле `traffic_used_mb` уже есть).
- **Ограничение отдачи (upload)** — добавить виртуальный интерфейс `ifb0` и зеркалить
  входящий трафик на него для шейпинга.
- **QR-код ключа** прямо в сообщении бота (библиотека `qrcode`).
- **Автопродление** и сохранение карты через эквайринг.
- **Резервные копии** файла базы `*.db` (просто копируй его по расписанию).

## 15. Коротко: порядок действий

1. `bash deploy/setup.sh`
2. Создать бота у @BotFather, узнать свой ID у @userinfobot
3. Узнать данные VPN-сервера (`wg show` / `awg show`, внешний IP, интерфейсы)
4. Заполнить `.env`
5. Запустить `python bot.py` в безопасном режиме и проверить весь круг покупки
6. Поставить `APPLY_TRAFFIC_RULES=true`, выполнить `setup_root_qdisc`, запустить от root
7. Запустить панель `python web_panel.py`
8. Настроить автозапуск через systemd

Удачи! Если что-то не получается — запусти `python bot.py` руками и внимательно прочитай
текст ошибки: там почти всегда написано, чего не хватает.
