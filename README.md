# Eshka — крипто-бот для KuCoin с ИИ

Бот, который сам торгует на бирже KuCoin: и спотом, и фьючерсами с плечом. Решения принимает по двум источникам: технический анализ свечей (RSI, скользящие средние) + ИИ-анализ новостей из интернета и Telegram-каналов.

> ⚠️ **Прочитай это**: торговля криптой — это риск. Любой бот может слить депозит. Сначала ВСЕГДА гоняй в режиме симуляции (LIVE_TRADING=false). Включай реальные деньги только когда сам понимаешь, что делаешь. Никто не несёт ответственности за твои потери, кроме тебя.

## Как это устроено (для человека, который только начал программировать)

Программа — это набор файлов. Каждый файл делает одну конкретную вещь. Когда ты запускаешь `bot.py`, он подтягивает остальные файлы как "помощников":

```
bot.py            ←  главный файл, запускает цикл
├── config.py     ←  все настройки (что торгуем, какие риски, ключи API)
├── news_collector.py  ←  собирает новости из RSS и Telegram
├── ai_analyzer.py     ←  отдаёт новости в ИИ и получает оценки
├── strategies.py      ←  считает индикаторы (RSI, MA) по свечам
├── trader.py          ←  решает, сколько денег вложить, и ставит стоп-лосс
└── kucoin_client.py   ←  разговаривает с биржей (получает цены, отправляет ордера)
```

Принцип "одна задача — один файл" называется **модульность**. Так гораздо проще:
- найти, где что-то поломалось;
- поменять одну часть, не сломав остальные.

### Логика бота простыми словами

Каждые 5 минут бот делает следующее:

1. **Идёт за новостями.** Скачивает свежие заголовки с CoinDesk, Cointelegraph и из твоих Telegram-каналов.
2. **Спрашивает ИИ.** Отправляет новости в Claude (модель от Anthropic) с вопросом: "Это позитив или негатив для BTC, ETH, SOL?". ИИ возвращает оценки.
3. **Смотрит на свечи.** Качает 100 последних часовых свечей с биржи, считает RSI и скользящие средние. Получает технический сигнал (buy / sell / hold).
4. **Принимает решение.** Если ИИ И техника согласны — действует. Если расходятся — пропускает. **Лучше ничего не сделать, чем сделать плохую сделку.**
5. **Если уже есть открытая позиция** — проверяет: не пора ли закрыть по стоп-лоссу (защита от убытка) или тейк-профиту (фиксация прибыли).

## Как запустить

### Шаг 1. Установи Python

Нужен Python 3.10 или новее. Проверь:
```bash
python3 --version
```

### Шаг 2. Скачай этот проект и установи зависимости

```bash
git clone <адрес-этого-репозитория>
cd Eshka
pip install -r requirements.txt
```

### Шаг 3. Получи ключи

**KuCoin API:**
1. Зарегистрируйся на [kucoin.com](https://www.kucoin.com).
2. Профиль → API Management → Create API.
3. Выдай права: `General` + `Trade`. **Withdraw НЕ ВЫДАВАЙ**, чтобы бот не мог вывести деньги, если ключи утекут.
4. Сохрани три значения: API Key, Secret, Passphrase.

**Anthropic (Claude):**
1. Создай аккаунт на [console.anthropic.com](https://console.anthropic.com).
2. Получи API-ключ (вкладка API Keys).
3. Закинь $5 на баланс — этого хватит на месяцы работы.

**Telegram (опционально):**
1. Зайди на [my.telegram.org](https://my.telegram.org/apps).
2. Создай приложение, получи `api_id` и `api_hash`.

### Шаг 4. Заполни .env

```bash
cp .env.example .env
nano .env   # или открой в любом редакторе
```

Вставь все ключи. **LIVE_TRADING оставь `false`!** Это режим симуляции, никаких реальных денег.

### Шаг 5. Запусти

```bash
python bot.py
```

Увидишь лог в консоли и в файле `bot.log`. Бот будет крутиться, пока не нажмёшь Ctrl+C.

## Как настраивать

Все настройки — в файле `config.py`. Открой его и поправь под себя:

| Параметр | Что значит | Совет новичку |
|---|---|---|
| `SPOT_SYMBOLS` | Какие пары торгуем на споте | Начни с одной — `BTC-USDT` |
| `FUTURES_SYMBOLS` | Какие фьючерсы торгуем | Поставь `[]` (пусто), пока не разберёшься |
| `RISK_PER_TRADE_PERCENT` | % депо на одну сделку | 1-2%, не больше |
| `STOP_LOSS_PERCENT` | Когда закрываем в минус | 3% — золотая середина |
| `TAKE_PROFIT_PERCENT` | Когда фиксируем прибыль | 6% (вдвое больше стопа — соотношение риск/прибыль 1:2) |
| `MAX_LEVERAGE` | Максимальное плечо на фьючерсах | 3x — для новичка это уже много |
| `TIMEFRAME` | Размер свечи | `1hour` — меньше шума, чем `5min` |
| `LOOP_INTERVAL_SECONDS` | Раз в сколько секунд проверять рынок | 300 (5 мин) |

## Стратегии — что под капотом

### RSI (индекс относительной силы)
Число от 0 до 100. Показывает, "перекуплен" или "перепродан" актив:
- **< 30** — перепродан, скорее всего отскочит вверх → **сигнал на покупку**
- **> 70** — перекуплен, скорее всего пойдёт коррекция → **сигнал на продажу**

Аналогия: если резинку растянули до предела (RSI=80), скорее всего она вернётся обратно.

### MA Crossover (пересечение средних)
Берём среднюю цену за 9 свечей (быстрая) и за 21 свечу (медленная):
- быстрая пересекла медленную **снизу вверх** → начался рост → **покупка**
- быстрая пересекла медленную **сверху вниз** → начался спад → **продажа**

Аналогия: быстрый бегун начал обгонять медленного — значит, движение ускоряется.

### ИИ-анализ новостей
Новости в текстовом виде сложно "посчитать". Но Claude хорошо понимает контекст:
- "SEC одобрила Bitcoin ETF" → ИИ ставит BTC `bullish, confidence=90`
- "Хакеры украли $200M с биржи Foo" → биржевые токены `bearish`

Бот доверяет ИИ только если `confidence >= 60`. Иначе игнорирует и слушает только технику.

### Объединение сигналов
- Техника говорит "buy" + ИИ говорит "bullish" → **покупаем**
- Техника "buy" + ИИ "bearish" → **держимся в стороне** (противоречие)
- ИИ молчит (нет новостей) → решаем по технике

## Что НЕ умеет этот бот (честно)

- **Не предсказывает чёрных лебедей.** Если завтра рухнет крупная биржа, бот узнает об этом, когда узнаешь ты — из новостей. К тому моменту цены уже улетят.
- **Не умеет грид-торговлю и арбитраж.** Это другие стратегии, их можно дописать модулями.
- **Не использует ордербук и стакан.** Решения принимаются только по свечам — для дневных сделок этого хватает, для скальпинга — нет.
- **Не учится сам.** ИИ не запоминает прошлые сделки. Если хочешь "обучаемого" бота — нужен отдельный модуль с историей сделок и анализом ошибок.

## Чек-лист перед запуском с реальными деньгами

1. ✅ Прогнал в симуляции минимум 2 недели и посмотрел `trades.json` — статистика положительная?
2. ✅ В KuCoin отключил у API-ключа право `Withdraw`.
3. ✅ `RISK_PER_TRADE_PERCENT` ≤ 2%.
4. ✅ Плечо на фьючерсах ≤ 3x (а лучше начать только со спота).
5. ✅ На депо лежит сумма, которую не страшно потерять полностью.
6. ✅ Понял, что делает каждый файл в проекте, и можешь объяснить вслух.

Если хотя бы один пункт "нет" — рано включать `LIVE_TRADING=true`.

- `bot.log` — все логи (можно открыть в любом редакторе).
- `trades.json` — журнал сделок (JSONL, по строке на сделку).
- `bot_session.session` — авторизация Telegram (создаётся один раз).

Все эти файлы в `.gitignore`, чтобы случайно не закоммитить.

## Если что-то сломалось

1. Открой `bot.log`, прокрути в конец — ошибка обычно написана прямым текстом.
2. Если `ImportError` — забыл установить зависимости: `pip install -r requirements.txt`.
3. Если `Invalid API key` — проверь `.env`, особенно `KUCOIN_API_PASSPHRASE` (его часто путают с паролем от аккаунта — это РАЗНЫЕ вещи).
4. Если ИИ не отвечает — проверь баланс на console.anthropic.com.

Удачи. И помни главное правило трейдера: **сохрани депозит. Прибыль — потом.**
